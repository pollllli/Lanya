# 器件管理系统 - 串口通信协议

## 概述

本协议定义了器件管理系统与下位机之间的串口通信规范。所有指令帧均采用同步通信方式，APP端发送指令后等待MCU端响应。

## 帧格式

### 通用帧结构（7字节）

```
字节序号:  0     1     2     3     4      5      6
          ↓     ↓     ↓     ↓     ↓      ↓      ↓
内容:   [帧头] [帧头] [命令] [长度] [数据高] [数据低] [校验]
        [55]  [AA]  [CMD] [LEN] [DATA_H] [DATA_L] [CRC]
```

### 字段说明

| 字段 | 长度 | 说明 |
|-----|-----|-----|
| 帧头 | 2字节 | 固定为 `0x55 AA` |
| 命令字 | 1字节 | 指令类型标识 |
| 数据长度 | 1字节 | 数据字段的字节数（固定为 `0x02`） |
| 数据 | 2字节 | uint16_t 类型，高字节在前，低字节在后 |
| CRC校验 | 1字节 | CRC-8/MAXIM 校验码 |

## 数据格式

数据字段采用 **uint16_t** 类型，表示范围：`0x0000` ~ `0xFFFF`

### 字节序

采用 **大端序（Big-Endian）**，即高字节在前，低字节在后：

```
数据值: 0xABCD
发送顺序: [0xAB] [0xCD]
```

## CRC-8/MAXIM 校验算法

### 算法参数

| 参数 | 值 |
|-----|-----|
| 多项式 | 0x31 (x^8 + x^5 + x^4 + 1) |
| 初始值 | 0x00 |
| 输入反转 | 是 (REFIN=true) |
| 输出反转 | 是 (REFOUT=true) |
| 异或输出 | 无 |

### 计算示例

以指令帧 `55 AA 01 02 00 01` 为例：

```
输入数据: [0x55, 0xAA, 0x01, 0x02, 0x00, 0x01]
CRC结果: 0xD5
完整帧: 55 AA 01 02 00 01 D5
```

## 响应帧规则

### 命令字映射

响应帧的命令字 = 发送帧命令字 + 0x80

| 发送命令字 | 响应命令字 |
|-----------|-----------|
| 0x00 | 0x80 |
| 0x01 | 0x81 |
| 0x02 | 0x82 |
| 0x03 | 0x83 |

### 响应数据

响应帧的数据字段应与发送帧的数据字段相同，表示操作已成功执行。

---

## 错误处理

### 校验错误

如果MCU端接收到的帧CRC校验失败，应忽略该帧，不发送响应。

### 未知命令

如果MCU端接收到未定义的命令字，应忽略该帧，不发送响应。

### 数据格式错误

如果数据长度或数据格式不符合规范，应忽略该帧，不发送响应。

## 附录：CRC-8/MAXIM 计算代码

```javascript
function reverseByte(byte) {
  let result = 0;
  for (let i = 0; i < 8; i++) {
    result = (result << 1) | ((byte >> i) & 1);
  }
  return result & 0xFF;
}

const CRC8_MAXIM_TABLE = (() => {
  const table = [];
  const polynomial = 0x31;
  for (let i = 0; i < 256; i++) {
    let crc = i;
    for (let j = 0; j < 8; j++) {
      crc = (crc & 0x80) ? ((crc << 1) ^ polynomial) : (crc << 1);
    }
    table[i] = crc & 0xFF;
  }
  return table;
})();

function calculateCRC8(data) {
  let crc = 0;
  for (let i = 0; i < data.length; i++) {
    const byte = reverseByte(data[i] & 0xFF);
    crc = CRC8_MAXIM_TABLE[(crc ^ byte) & 0xFF];
  }
  return reverseByte(crc) & 0xFF;
}
```
